The plugin features a robust, multi-tier GUI system for a seamless player experience.

  • Slot 0 — a button rendered as your own player head, named "Player Identity". Opens the Player Identity GUI.

Player Identity GUI dynamic · paginated

Opened from Slot 0 of the Main Identity GUI. It lists every base "player" identity you own — the always-available Primary identity plus every unlocked Alternative identity (see Key Features) — and lets you switch between them.

  • Title: Player Identity - Page {page number}.
  • Alternative identities: listed from Slot 0 upward (index 1…n), up to 45 per page across the content rows. Each is shown as a head textured with a texture picked at random from its own profession's available textures (stable per identity). Clicking an alternative identity head equips that identity.
  • Dynamic sizing: the content area scales with your unlocked identities — 9 alternatives need one content row, 10 need two, and so on. A bottom control row is always appended below the content.
  • Primary identity: shown as your own player head at the center of the control row. On any multi-page view the GUI uses the full six rows, placing the primary identity permanently at Slot 49. Clicking it equips the Primary identity.
  • Equipped marker: the currently equipped base identity shows "Currently equipped." in its lore; every other head shows "Click to equip."
  • Back button: the first slot of the control row (Slot 45 on six-row views) returns to the Main Identity GUI.
  • Pagination: on six-row views, Slot 48 is "Previous Page" and Slot 50 is "Next Page". Each button appears only when the corresponding page exists.
  • Switching rules: switching here works between base identities only — a regular profession identity must be unequipped first so its key item is handed back. Each identity keeps its own isolated inventory and ender chest, saved and restored on every switch, and the identity swap cooldown applies.

Cosmetic GUI 1-row

Provides a menu to manage visual aspects of the identity:

  • Slot 5 — displays a Skin placeholder.
  • Slot 7 — displays an Aura placeholder.
  • Security: this menu is only accessible if the player has an active identity equipped.

Ability & Rune GUIs 6-rows

These GUIs are dynamically bound to the database:

Equipped (Lime Stained Glass)Empty (Red Stained Glass)
  • Fetching: upon opening, the plugin fetches the currently equipped abilities / runes for the active identity.
  • Placeholders: empty slots are shown as Red Stained Glass Panes.
  • Equipped items: equipped abilities / runes are shown as Lime Stained Glass Panes with their respective display names.
  • Security: these menus are only accessible if the player has an active identity equipped.
