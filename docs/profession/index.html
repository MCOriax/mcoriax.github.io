How profession EXP works

Server owners customise how professions gain experience by defining action-based EXP configurations in YAML files within the professions/ folder. New professions are registered into the database automatically on first run.

Key requirements

  • system_code — must be unique and provided in lowercase.
  • display_name — the human-readable name for the profession.

EXP gain syntax

Simple value

Award a specific amount or random range of EXP.

receive_damage: 10 # Fixed
deal_damage: 10~20 # Random range
Condition-based (numerical)

Award EXP based on numerical thresholds (e.g. damage amount). Supports range keys (min-max or min~max).

deal_damage:
  "1-10": 5       # 1 to 10 damage awards 5 EXP
  "11-50": 10~20  # 11 to 50 damage awards random 10-20 EXP
  "51-100": 50    # 51 to 100 damage awards 50 EXP
Condition-based (string)

Award EXP based on specific targets (e.g. mob types).

kill_mob:
  zombie: 20      # Killing a zombie awards 20 EXP
  skeleton: 25    # Killing a skeleton awards 25 EXP
  ender_dragon: 1000~2000 # Boss kill awards random large EXP
Configuration keys
  • level.max / level.exp_up — the max level and EXP required per level.
  • stats — starting attributes (energy, health, food, strength, defense, wisdom, luck, agility, resilience).
  • stats_random — bonus points randomly distributed across the six primary attributes on creation.
  • skins — available Base64 skin textures; one is chosen at random per new identity.
ℹ️

Editing or deleting a default file does not bring it back automatically; defaults are only written when the file is missing.
