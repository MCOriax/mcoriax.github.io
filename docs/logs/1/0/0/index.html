Change Log — 1.0.0

Permalink for version 1.0.0.

1.0.0 Latest 2026-07-17 · Initial release

The first public release of MCIdentity — a robust, high-performance identity management system for the Bukkit family of servers.

Added Core identity system

  • Primary base player identity per player, plus unlockable alternative base identities of any other profession.
  • Profession identity keys as in-game items with custom colored display names; unbound until first used.
  • Per-identity isolated inventory, ender chest, level, and stats — switching identities is a full context switch.
  • Passive base leveling on two tracks (primary 86,400 EXP/level, alternative 3,600 EXP/level), buffered and flushed in bulk.

Added Platforms & storage

  • Full support for SpigotMC, PaperMC, and Folia, including Folia's regionalized scheduling.
  • Selectable storage backends: SQLite (default), MySQL, PostgreSQL, MongoDB, and the MCIdentity central REST server.
  • Fully asynchronous storage access with RAM caching, safe fallbacks, and validated stat identifiers.
  • Cold-boot login protection for the central server backend.

Added Gameplay & GUIs

  • Multi-tier GUI system: Main Identity, paginated Player Identity, Cosmetic, Ability, and Rune menus.
  • 21 bundled professions with action-based EXP configuration and anti-farm safeguards.
  • Native MythicMobs ability integration (ability code + skin model).
  • Role-based command permissions with the mcidentity.admin node.

Developer Public API & events

  • MCIdentityProvider published to GitHub Packages (api, common, bukkit, version 1.0.0).
  • Custom cancellable Bukkit events for equip, unequip, creation, death, rune/ability/skin/aura set, and stat add.
