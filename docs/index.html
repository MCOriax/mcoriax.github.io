What MCIdentity does

Every player has exactly one primary identity — their permanent base player profession, keyed by their own UUID. Beyond it, players can unlock multiple alternative base identities, each of any other profession, plus equip profession identity keys found in-game. Each identity keeps its own isolated inventory, ender chest, level, and stats, and switching is a full context switch — as if logging into a completely different account.

The whole system is built for performance and safety: identity data can live in SQLite (the zero-configuration default), MySQL, PostgreSQL, or MongoDB, or be owned by the MCIdentity central server and accessed through asynchronous REST calls. Every backend is combined with RAM caching and fully asynchronous access for a lag-free experience on high-population servers.
