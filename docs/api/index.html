Developer API Usage

This guide explains how to integrate a new custom plugin (for example an economy, bank, or membership system) with MCIdentity. The identity system is exposed through the MCIdentityProvider and the Bukkit event layer, all published to GitHub Packages.

📡

Looking for the full event catalog? See Events & Listeners.

1 · Dependency import

MCIdentity's library modules are published to the GitHub Packages Maven registry for the repository MCOriax/mcidentity. GitHub Packages always requires authentication, even to read public packages, so consumers must supply a GitHub username and a Personal Access Token (classic) with the read:packages scope.

1.1 · Configure the repository and credentials

Never hard-code credentials in build.gradle. Read them from environment variables or from your ~/.gradle/gradle.properties (outside the project):

~/.gradle/gradle.properties 
gpr.user=your-github-username
gpr.key=ghp_your_personal_access_token_with_read_packages
build.gradle 
repositories {
    mavenCentral()

    maven {
        name = "MCIdentityGitHubPackages"
        url = uri("https://maven.pkg.github.com/MCOriax/mcidentity")
        credentials {
            // Falls back to the GITHUB_ACTOR / GITHUB_TOKEN env vars in CI.
            username = project.findProperty("gpr.user") ?: System.getenv("GITHUB_ACTOR")
            password = project.findProperty("gpr.key")  ?: System.getenv("GITHUB_TOKEN")
        }
    }
}

1.2 · Declare the three dependencies compileOnly · 2.0.0

MCIdentity is present on the server at runtime, so you compile against it but do not bundle/shade it — use the compileOnly configuration for all three artifacts. The published coordinates are group io.github.mcoriax, version 2.0.0:

build.gradle 
dependencies {
    // MCIdentity — provided by the server at runtime, so compileOnly only.
    compileOnly 'io.github.mcoriax:api:2.0.0'      // interfaces & models
    compileOnly 'io.github.mcoriax:common:2.0.0'   // MCIdentityProvider + data layer
    compileOnly 'io.github.mcoriax:bukkit:2.0.0'   // Bukkit events & helpers

    // Your own server platform API (example)
    compileOnly 'org.spigotmc:spigot-api:1.20.1-R0.1-SNAPSHOT'
}
ℹ️

The jar file names are mcidentity-api-2.0.0.jar, mcidentity-common-2.0.0.jar, and mcidentity-bukkit-2.0.0.jar, but the resolvable Maven artifactIds are api, common, and bukkit as shown above.

1.3 · Load order

So MCIdentity initializes before your plugin, declare it as a dependency in your own plugin.yml:

plugin.yml 
name: MyBankPlugin
main: com.example.bank.BankPlugin
version: 1.0.0
api-version: '1.20'
depend: [MCIdentity]        # use softdepend if MCIdentity is optional

2 · API usage

All identity data flows through io.github.mcoriax.mcidentity.common.MCIdentityProvider, a singleton exposed as a public static field and initialized by MCIdentity when its database comes up.

import io.github.mcoriax.mcidentity.common.MCIdentityProvider;

if (!MCIdentityProvider.hasDatabase()) {
    // MCIdentity has no database configured yet — do not touch the API.
    return;
}
MCIdentityProvider provider = MCIdentityProvider.instance;
🧵

Threading. Every database operation returns a CompletableFuture<…> and completes on an asynchronous thread. Do the identity work inside the future's callbacks, but hop back to the server thread (Bukkit scheduler, or the region/global scheduler on Folia) before calling any thread-confined Bukkit API such as inventories or entities.

2.1 · Fetch — read the active identity's data

getActiveIdentity(playerUuid) resolves the identity UUID the player is currently playing as; the per-identity data is then keyed by that identity UUID.

import io.github.mcoriax.mcidentity.api.model.IdentityProperties;
import java.util.UUID;

UUID playerUuid = player.getUniqueId();

provider.getActiveIdentity(playerUuid).thenAccept(optIdentity -> {
    if (optIdentity.isEmpty()) {
        return;                          // player has no identity equipped
    }
    UUID identityUuid = optIdentity.get();

    provider.getProperties(identityUuid).thenAccept(optProps ->
        optProps.ifPresent(props -> {
            int level = props.getLevel();
            int exp   = props.getExp();
            // ... react to this identity's level/exp
        }));

    // A single stat value (see StatKeys for the allow-list, e.g. "strength"):
    provider.getStat(identityUuid, "strength")
            .thenAccept(strength -> { /* ... */ });
});

The full session (player UUID, active identity UUID, serialized key item) is available via getSession(playerUuid):

import io.github.mcoriax.mcidentity.api.model.IdentitySession;

provider.getSession(playerUuid).thenAccept(optSession ->
    optSession.ifPresent(session -> {
        UUID identityUuid = session.getIdentityUniqueId();
        String contentKey = session.getContentKey();
    }));

2.2 · Modify — persist changes to an identity

IdentityProperties values are snapshots. For cosmetic/storage fields use MutableIdentityProperties.copyOf(...) and its setters; for level, exp, or statPoints, build a fresh value and call updateProperties:

import io.github.mcoriax.mcidentity.common.model.MutableIdentityProperties;

provider.getProperties(identityUuid).thenCompose(optProps -> {
    if (optProps.isEmpty()) {
        return java.util.concurrent.CompletableFuture.completedFuture(null);
    }
    IdentityProperties current = optProps.get();

    // Level up this identity, carrying the remaining fields over unchanged.
    MutableIdentityProperties leveled = new MutableIdentityProperties(
        current.getLevel() + 1,   // level
        0,                        // exp (reset)
        current.getStatPoints() + 5,
        current.getHeadText(),
        current.getSkin(),
        current.getAura(),
        current.getInventoryContents(),
        current.getEnderchestContents()
    );
    return provider.updateProperties(identityUuid, leveled);
});

// Atomic stat change (safe under concurrency), negative to subtract:
provider.addStat(identityUuid, "defense", 10);

// Cosmetics:
provider.saveSkin(identityUuid, base64SkinTexture);
provider.saveAura(identityUuid, "aura_flame");

// Per-identity storage — the inventory and ender chest are isolated per
// identity. Pass null for a storage you do not want to touch.
provider.updateIdentityStorage(identityUuid, base64Inventory, base64EnderChest);
provider.getIdentityInventory(identityUuid).thenAccept(base64 -> { /* restore inventory */ });
provider.getIdentityEnderchest(identityUuid).thenAccept(base64 -> { /* restore ender chest */ });

2.3 · Manage — active identity, actions, and cooldowns

import io.github.mcoriax.mcidentity.api.model.ActionType;

// Set (or clear, with null) the player's active identity + serialized key item.
provider.setActiveIdentity(playerUuid, identityUuid, base64KeyItem);

// Audit log — ActionType is EQUIP, UNEQUIP, DEATH_DROP, or ADMIN_REMOVE.
provider.logAction(playerUuid, identityUuid, ActionType.EQUIP);

// Swap cooldown helpers (RAM-only, synchronous):
if (!provider.isOnSwapCooldown(playerUuid)) {
    provider.setSwapCooldown(playerUuid);
}

2.4 · Check the active profession — region whitelist / blacklist

getActiveProfession(playerUuid) returns the profession system code of the player's currently active identity. Unlike the data getters above it is synchronous and RAM-only (backed by the active session cache), so it is safe to call on hot paths such as a region-entry check. It returns an empty Optional when the player has no active session cached (offline, or between identity swaps).

import java.util.Optional;
import java.util.Set;

// Only these professions may enter the guarded region (whitelist).
private static final Set<String> ALLOWED = Set.of("guard", "fighter", "archer");

boolean mayEnter(UUID playerUuid) {
    Optional<String> profession = provider.getActiveProfession(playerUuid);
    // No active identity → deny (choose your own default for the "no identity" case).
    return profession.map(code -> ALLOWED.contains(code.toLowerCase()))
                     .orElse(false);
}

// Blacklist variant — everyone except merchants may enter:
private static final Set<String> DENIED = Set.of("merchant");

boolean isBlocked(UUID playerUuid) {
    return provider.getActiveProfession(playerUuid)
                   .map(code -> DENIED.contains(code.toLowerCase()))
                   .orElse(false);
}
🔤

Profession codes are stored lower-cased; compare case-insensitively as shown. For an authoritative/offline lookup instead of the RAM cache, chain getActiveIdentity(playerUuid) → getIdentityKey(identityUuid) → getProfessionCode().

3 · Bukkit integration

MCIdentity fires custom Bukkit events from the io.github.mcoriax.mcidentity.bukkit.event package. Every event works across SpigotMC, PaperMC, and Folia. The two events most relevant to an economy/membership plugin are MCIdentityEquipEvent and MCIdentityUnequipEvent — the exact moments to load and unload the data bound to an identity.

3.1 · Implement a listener

Your own plugin 
package com.example.bank;

import io.github.mcoriax.mcidentity.bukkit.event.MCIdentityEquipEvent;
import io.github.mcoriax.mcidentity.bukkit.event.MCIdentityUnequipEvent;
import org.bukkit.entity.Player;
import org.bukkit.event.EventHandler;
import org.bukkit.event.EventPriority;
import org.bukkit.event.Listener;

import java.util.UUID;

public class IdentityBankListener implements Listener {

    private final BankService bank;

    public IdentityBankListener(BankService bank) {
        this.bank = bank;
    }

    // Fired when a player equips an identity. Load THAT identity's account.
    @EventHandler(priority = EventPriority.NORMAL, ignoreCancelled = true)
    public void onEquip(MCIdentityEquipEvent event) {
        Player player       = event.getPlayer();
        UUID   identityUuid = event.getIdentityUuid();   // bind data to THIS uuid
        // event.getIdentityItem() is the key ItemStack being equipped.

        bank.loadAccount(identityUuid);
    }

    // Fired when a player unequips an identity. Persist & unload its account.
    @EventHandler(priority = EventPriority.NORMAL, ignoreCancelled = true)
    public void onUnequip(MCIdentityUnequipEvent event) {
        Player player       = event.getPlayer();
        UUID   identityUuid = event.getIdentityUuid();

        bank.saveAndUnload(identityUuid);
    }
}

Register it in your plugin's onEnable:

getServer().getPluginManager().registerEvents(
        new IdentityBankListener(bankService), this);

Cancelling. Both events implement Cancellable. Calling event.setCancelled(true) vetoes the equip/unequip — useful if, for example, a membership must be active before an identity can be worn.

Folia / async safety. MCIdentity events mark themselves asynchronous whenever fired off the primary thread. Keep your handlers thread-safe and, when you must call main-thread-only Bukkit API, schedule back onto the appropriate scheduler instead of calling it directly.

The complete catalog — creation, death, rune/ability/skin/aura set, stat add, and the two events above — lives on the Events & Listeners page.

4 · Architecture warning — one identity is one account

⚠️

Each identity is an entirely separate character. Bind your data to the Identity UUID, never to the base Player UUID.

A single player owns many identities, and each identity has its own distinct UUID (its IdentityKey). Only one identity is "active" at a time, and switching identities is a full context switch — as if the player logged into a completely different account. Treat every identity UUID as a separate account.

Do this — key data by the Identity UUID

// CORRECT: the balance belongs to the identity, not the human behind it.
UUID identityUuid = event.getIdentityUuid();          // from equip/unequip events
bank.setBalance(identityUuid, 1000);

Never do this — keying by the Player UUID

// WRONG: every identity this player owns would share ONE balance,
// membership, or rank — collapsing distinct characters into one account.
bank.setBalance(player.getUniqueId(), 1000);

Consequences of keying by Player#getUniqueId():

  • Bank balances, membership statuses, ranks, and cooldowns leak across characters that are meant to be independent.
  • Money or perks earned on one identity appear on all of the player's other identities.
  • Per-character progression and economy balancing become impossible.

Rules for third-party plugins:

  1. Resolve the current identity with MCIdentityProvider.instance.getActiveIdentity(playerUuid) (or read event.getIdentityUuid() from the events) and use that UUID as the primary key for all of your stored data.
  2. Load a character's data on MCIdentityEquipEvent and flush/unload it on MCIdentityUnequipEvent.
  3. Even the default state is its own identity with its own UUID — there is no "no identity" account to fall back to the player UUID for. If no identity is active, hold your operation until one is equipped.

Full MCIdentityProvider method reference

The public methods you can call on MCIdentityProvider.instance. Async methods return a CompletableFuture; RAM-only helpers are synchronous.

Registry
createProfession(String systemCode, String displayName)

Creates a new profession type in the database. systemCode is the unique identifier; displayName is the human-readable name.

Returns CompletableFuture<Void>

createAbility(String abilityCode, String displayName, String abilitySkin)

Creates a new ability type. abilityCode maps to the MythicMobs ability ID; abilitySkin is the model ID rendered for the ability, or null for the default skin.

Returns CompletableFuture<Void>

getProfessions()

Retrieves all available professions.

Returns CompletableFuture<Collection<Profession>>

getAbilities()

Retrieves all available abilities.

Returns CompletableFuture<Collection<Ability>>

getRunes()

Retrieves all available runes.

Returns CompletableFuture<Collection<Rune>>

Identities & properties
createIdentity(IdentityKey key, IdentityProperties properties, IdentityStats stats)

Creates a new identity record with initial properties and stats. key is the identity key (UUID, Profession); properties holds level, exp, etc.; stats holds health, energy, strength, etc.

Returns CompletableFuture<Void>

getIdentityKey(UUID uuid)

Retrieves an identity key by its unique UUID.

Returns CompletableFuture<Optional<IdentityKey>>

getProperties(UUID identityUuid)

Retrieves current properties (level, exp, etc.) for an identity.

Returns CompletableFuture<Optional<IdentityProperties>>

updateProperties(UUID identityUuid, IdentityProperties properties)

Persists updated properties to the database and refreshes the RAM cache if active.

Returns CompletableFuture<Void>

Stats
getStats(UUID identityUuid)

Retrieves current stats (health, energy, etc.) for an identity.

Returns CompletableFuture<Optional<IdentityStats>>

getStat(UUID identityUuid, String statKey)

Retrieves the value of a specific stat. statKey is the internal code (e.g. "strength").

Returns CompletableFuture<Integer>

updateStats(UUID identityUuid, IdentityStats stats)

Persists updated stats to the database and refreshes the RAM cache if active.

Returns CompletableFuture<Void>

addStat(UUID identityUuid, String statKey, int amount)

Atomically increments a specific stat in the database. amount can be negative.

Returns CompletableFuture<Void>

Abilities & runes (equipped)
getEquippedAbilities(UUID identityUuid)

Retrieves a map of equipped abilities.

Returns CompletableFuture<Map<Integer, EquippedAbility>> (slot → abilityCode + abilitySkin)

setEquippedAbility(UUID identityUuid, int slot, String abilityCode, String abilitySkin)

Equips or clears an ability in a specific slot. abilityCode is the MythicMobs ability ID (null to clear); abilitySkin is the skin model ID stored with the slot, or null for the default skin.

Returns CompletableFuture<Void>

getRunes(UUID identityUuid)

Retrieves a map of equipped runes.

Returns CompletableFuture<Map<Integer, String>> (slot → rune code)

setRune(UUID identityUuid, int slot, String runeCode)

Sets or clears a rune in a specific slot (null to clear).

Returns CompletableFuture<Void>

Active identity & session
getActiveIdentity(UUID playerUuid)

Gets the UUID of the identity currently equipped by a player.

Returns CompletableFuture<Optional<UUID>>

getActiveProfession(UUID playerUuid)

Gets the profession system code of the player's currently active identity. Synchronous and RAM-only (reads the active session cache), so it is safe on hot paths such as region profession whitelist/blacklist checks. Empty when no active session is cached.

Returns Optional<String>

getSession(UUID playerUuid)

Retrieves full session info for a player (identity UUID and serialized item content).

Returns CompletableFuture<Optional<IdentitySession>>

setActiveIdentity(UUID playerUuid, UUID identityUuid, String contentKey)

Sets a player's active identity and manages session caching. identityUuid null to unequip; contentKey is the Base64 serialized item stack of the identity key.

Returns CompletableFuture<Void>

updateSessionContentKey(UUID playerUuid, String contentKey)

Rewrites only the serialized identity item stored on a player's active session, leaving the equipped identity unchanged, and refreshes the RAM cache if active.

Returns CompletableFuture<Void>

logAction(UUID playerUuid, UUID identityUuid, ActionType actionType)

Logs an identity-related action (EQUIP, UNEQUIP, etc.) to the database.

Returns CompletableFuture<Void>

Cosmetics
getCosmetics(UUID uuid)

Retrieves cosmetics (skin, aura) for an identity.

Returns CompletableFuture<Map<String, String>>

saveSkin(UUID uuid, String skin)

Saves the skin cosmetic (Base64 skin texture) for an identity.

Returns CompletableFuture<Void>

saveAura(UUID uuid, String aura)

Saves the aura cosmetic (aura system code) for an identity.

Returns CompletableFuture<Void>

Per-identity storage
getIdentityInventory(UUID identityUuid)

Reads the serialized inventory contents stored for an identity. Each identity keeps its own isolated inventory.

Returns CompletableFuture<String> (Base64 inventory, or null when unset)

getIdentityEnderchest(UUID identityUuid)

Reads the serialized ender chest contents stored for an identity. Each identity keeps its own isolated ender chest.

Returns CompletableFuture<String> (Base64 ender chest, or null when unset)

updateIdentityStorage(UUID identityUuid, String inventoryContents, String enderchestContents)

Persists the inventory and/or ender chest contents for an identity, carrying the other properties over unchanged. Pass null for a storage to leave it untouched.
