Commands & Permissions

Every /identity command, what it does, and the permission it needs.

Player commands

/identity

Open the main Identity GUI (1-row compact menu).

/identity help

Show the help message with all available commands.

/identity wiki

Send clickable links to the project wiki, one per line (https://mcoriax.github.io and https://mcoriax.gitlab.io); clicking a line opens it in your browser.

/identity check

Inspect the identity key in your hand or your currently equipped identity.

/identity unuse

Unequip your current identity.

/identity stats

View your remaining unspent stat points. A shortcut that behaves exactly like /identity stats get.

/identity stats get

View your remaining unspent stat points.

/identity stats add <stat_type> <amount>

Spend your stat points to upgrade specific attributes. Available stats are energy, health, food, strength, defense, wisdom, luck, agility, and resilience (e.g. /identity stats add food 5). Tab completion suggests every stat, including food.

Admin commands mcidentity.admin

/identity ability get <system code>

Admin tool to get an ability item. You can then give this to a player.

/identity ability create <ability_code_id> <ability_skin_id> <head_texture_base64> [custom_model_id]

Admin tool to create an equippable ability item.

  • ability_code_id — the MythicMobs ability (skill) ID the item represents.
  • ability_skin_id — the model ID rendered for the ability. Players equipping the item store this skin alongside the ability code.
  • head_texture_base64 — the Base64 profile-property texture applied to the player-head item.
  • custom_model_id (optional) — a numeric CustomModelData value for resource-pack models. Omit it or pass null to keep the default model.
  • Both IDs are saved in the item's persistent data (NBT), so equipping the item through the Ability GUI records exactly which MythicMobs ability and skin to use.
/identity rune get <system code>

Admin tool to get a rune item.

/identity get <profession|random> [texture]

Admin tool to give a new identity key. Provide a profession code or random to have one assigned upon first use. You can optionally provide a Base64 texture string.

/identity base add <player> <amount>

Grant a player additional alternative base identities, assigning each new slot a random non-player profession. For example, /identity base add MCOriax 2 adds 2 alternative identity slots, each a random class.

/identity base add <player> <profession> <amount>

The same grant, but every new slot is assigned the specific <profession> you name (e.g. /identity base add MCOriax warrior 3). The player profession cannot be added as a sub-identity — it is reserved for the primary identity, so the command is rejected and the sender is notified. The slot count is stored in the database and the new identities are registered immediately under deterministic UUIDs, so they appear in the Player Identity GUI right away.

Permissions

Standard player commands
Commands like /identity, /identity stats, and /identity unuse are available to all players by default. They do not require any permission nodes.
mcidentity.admin
Required for all administrative sub-commands, including /identity ability get, /identity ability create, and /identity base.
