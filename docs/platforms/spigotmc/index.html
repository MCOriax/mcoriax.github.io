MCIdentity on SpigotMC

The reference platform. SpigotMC provides the classic Bukkit API and the single-main-thread model MCIdentity's scheduler abstraction targets.

SpigotMC PaperMC FoliaMC

How it runs

The platforms:spigotmc module supplies a concrete plugin entry point that extends MCIdentity's shared Bukkit core. On Spigot, all gameplay code runs on the single main server thread, and the plugin's PlatformScheduler abstraction maps directly onto the standard BukkitScheduler.

  • Commands (/identity), tab completion, GUIs, and the EXP-gain listeners all use the standard Bukkit event and scheduling model.
  • Every database operation still runs asynchronously on a dedicated worker pool, so the main thread is never blocked by storage I/O.
  • Custom MCIdentity events are fired on the main thread and behave as ordinary synchronous Bukkit events.

Storage on Spigot

The SQLite and MySQL JDBC drivers ship with SpigotMC, so those backends work with no extra downloads. The PostgreSQL driver and the MongoDB client are packaged inside the plugin JAR. See Data Storage for configuration.

Requirements

  • Java 25 or newer.
  • A SpigotMC server on the 26.1.x API line.
