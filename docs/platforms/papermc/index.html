MCIdentity on PaperMC

Paper is a high-performance fork of Spigot. MCIdentity runs on it out of the box, with the same behaviour as Spigot plus Paper's performance improvements.

SpigotMC PaperMC FoliaMC

How it runs

The platforms:papermc module supplies a dedicated PaperMC entry point extending MCIdentity's shared Bukkit core. Because Paper is Bukkit-compatible and keeps the single main-thread model, the plugin's PlatformScheduler abstraction behaves just as it does on Spigot — while your server benefits from Paper's optimizations.

  • Commands, tab completion, GUIs, and the modular EXP-gain listeners all use the standard Bukkit event and scheduling model.
  • All storage I/O runs asynchronously on a dedicated worker pool, keeping the main thread free.
  • Custom MCIdentity events are fired on the main thread and observed like ordinary Bukkit events.

Storage on Paper

As on Spigot, the SQLite and MySQL JDBC drivers ship with the server; the PostgreSQL driver and MongoDB client are bundled in the plugin JAR. Configure your backend in Data Storage.

Requirements

  • Java 25 or newer.
  • A PaperMC server on the 26.1.x API line.
